# OW Ban Pick 设置、权限与比赛流程说明

本文档对应当前房间制版本，说明每项比赛设置会开启什么功能、各入口可以进行哪些操作，以及一场比赛从创建房间到结束的完整流程。

## 1. 房间入口与权限

每个房间创建后会生成四个独立入口。入口代码只用于鉴权，不应在公开页面中互换。

| 入口 | 主要权限 | 不可执行的操作 |
| --- | --- | --- |
| 队伍1 | 设置自己的赛前准备状态；在轮到本队时选图、填写本队上场成员、选择禁用先后或英雄；按比分规则提交/确认比分；暂停/恢复本队计时 | 修改房间配置；填写对方阵容；操作对方暂停；强制开始；回退检查点；管理员超时裁决 |
| 队伍2 | 与队伍1相同，但只操作队伍2一侧 | 与队伍1相同 |
| 房间管理员 | 编辑、确认和锁定房间配置；查看并代替双方执行比赛操作；普通开始或强制开始；处理超时；全局暂停；控制任意一队暂停；回退检查点 | 无法代替全局管理员管理其他房间或网站默认模板 |
| 直播 | 只读查看比赛状态、地图、阵容、英雄禁用、比分、倒计时和连接状态 | 所有会修改比赛状态的操作 |

连接状态条始终位于比赛主页面地图列表下方，赛前和比赛中都会显示。左侧显示队伍1的实际名称，中间显示管理员，右侧显示队伍2的实际名称；绿色代表已连接，红色代表已断开。赛前还会额外显示两队是否准备。

连接状态由短周期心跳维护。关闭页面或网络中断后，状态会在心跳有效期结束后变为断开；重新打开正确入口会恢复为已连接。

## 2. 配置生命周期

房间配置依次经历以下状态：

1. **待配置（draft）**：管理员可以导入模板、编辑并保存草稿。
2. **等待双方准备（ready）**：管理员已确认配置，但比赛尚未开始。两队可以确认队名并选择“是”完成准备。
3. **比赛进行（locked）**：配置锁定，普通编辑入口不可用。
4. **比赛结束（completed）**：达到赛制所需胜图数，停止创建后续地图流程。

管理员可以使用“回退到赛前配置”重新打开配置。此操作会清空地图、阵容、英雄禁用、比分和比赛进度，并重置两队准备状态，因此属于破坏性操作，页面会二次询问。

### 2.1 开始方式

| 条件 | 结果 |
| --- | --- |
| 两队均已准备，管理员点击“开始比赛” | 正常开始并锁定配置 |
| 任一队未准备，管理员点击普通开始 | 按钮不可用；后端也会拒绝开始请求 |
| 任一队未准备，管理员点击“管理员强制开始” | 立即开始并记录强制开始事件 |
| “以默认配置开始”为“是”，且两队均已准备 | 不需要管理员再次确认，自动锁定当前房间配置并开始 |
| “以默认配置开始”为“否”，且两队均已准备 | 仍等待管理员点击普通开始 |

管理员确认配置后，两队入口会在屏幕正中央显示赛前准备弹窗。若“以默认配置开始”为“是”，开关会立即保存，两队无需等待管理员确认即可进入准备。准备动作只有“是”，一旦确认不能撤销；配置变化会由管理员重新开启准备状态。配置仍不可准备时显示“管理员尚未确认配置”。

## 3. 所有比赛设置

### 3.1 比赛信息

| 设置 | 可选值/格式 | 对应功能 |
| --- | --- | --- |
| 比赛名称 | 1–120 个字符 | 显示在房间主标题和比赛状态中 |
| 队伍1名称 | 1–60 个字符 | 用于左侧队名、操作提示和连接状态，不额外添加“队伍1”前缀 |
| 队伍2名称 | 1–60 个字符 | 用于右侧队名、操作提示和连接状态，不额外添加“队伍2”前缀 |
| 队伍能否修改自己队伍的名称 | 是 / 否 | 为“是”时，各队可在准备弹窗中直接修改队名，点击“是”会以一次原子操作同时确认队名和准备；另一队的状态轮询不会覆盖尚未提交的队名草稿。为“否”时仍显示队名输入框，但为灰色禁用状态 |
| 比赛赛制 | FT2 / FT3 / FT4 | 分别先赢 2、3、4 张地图；准备 5、7、9 个地图槽位，以容纳最多两场平局 |
| 以默认配置开始 | 是 / 否 | 为“是”时，两队均准备后自动开始；为“否”时等待管理员开始 |

地图槽位不会一次全部展示：FT2 默认显示 3 张、FT3 默认显示 5 张、FT4 默认显示 7 张；只有默认最后一张仍未决出系列赛胜负时，才展示额外的第 4–5、6–7 或 8–9 张。

### 3.2 时间设置

| 设置 | 范围 | 生效阶段 |
| --- | --- | --- |
| 准备时间 `preStartRestSeconds` | 0–3600 秒 | 比赛开始后的赛前准备；0 表示跳过 |
| 地图选择时间 `mapSelectSeconds` | 1–3600 秒 | 每次需要队伍选择地图时 |
| 上场成员选择时间 `playerSelectSeconds` | 1–3600 秒 | 双方填写与确认本图阵容时 |
| 先后禁用选择时间 `firstBanChoiceSeconds` | 1–3600 秒 | 仅“败者选择先后”时，用于选择先手或后手 |
| 先手禁用时间 `firstBanActionSeconds` | 1–3600 秒 | 先手选择禁用英雄时 |
| 后手禁用时间 `secondBanActionSeconds` | 1–3600 秒 | 后手选择禁用英雄时 |
| 比分确认时间 `scoreConfirmSeconds` | 1–3600 秒 | 一方提交比分后，另一方确认时 |
| 场间时间 `postMatchRestSeconds` | 0–3600 秒 | 每张地图结束后；0 表示直接进入下一张 |

全局暂停会冻结当前活动倒计时。比分阶段的队伍暂停也会冻结比分确认倒计时；两队暂停重叠时只冻结一次，不会重复增加被扣除时间。

### 3.3 地图设置

#### 地图选择模式

| 值 | 行为 |
| --- | --- |
| 任意选择，地图不重复 `unique_map` | 允许选择任意合法模式，但不能重复已使用地图 |
| 模式轮换 `unique_mode_until_cycle` | 所有可用模式完成一轮前不重复模式 |
| 首图指定模式 `first_mode_then_unique_mode` | 第一张必须来自“第一张地图模式”，之后按模式不重复轮换 |
| 指定模式顺序 `strict_mode_order` | 每个地图槽位严格使用“模式顺序”中对应的模式 |
| 固定地图顺序 `fixed_map_order` | 按固定地图列表推进，不进入常规选图；固定第一张地图开关自动失效 |

相关设置：

| 设置 | 何时可用 | 对应功能 |
| --- | --- | --- |
| 地图池 | 所有模式 | 决定比赛中可被选择的地图；未勾选地图不可用 |
| 第一张地图模式 | 首图指定模式 | 限制第一张地图的模式 |
| 模式顺序 | 指定模式顺序 | 至少覆盖赛制的地图槽位数；每一项决定对应槽位模式 |
| 固定地图顺序 | 固定地图顺序模式 | 每个槽位直接使用指定地图 |
| 固定第一张地图 | 非固定地图顺序模式 | 开启后第一张图直接使用选择框中的地图，跳过第一张图选图阶段 |
| 第一张地图选择方 | 未固定第一张地图时 | 系统随机、交互随机、队伍1或队伍2 |
| 后续地图选择方 | 固定为败者选择 | 从第二张图起，由最近一张非平局地图的败者选择 |
| 对称地图允许选择阵营 | 是 / 否 | 为“是”时，占领、机动推进、闪点等对称模式也进入阵营选择；为“否”时只在攻击/护送和运载目标地图选择攻防 |
| 攻防选择方 | 上一张胜者 / 上一张败者 / 首次随机胜者 / 交互随机 / 系统随机 | 决定谁为本张地图选择进攻、防守或蓝红阵营；第一张图没有上一张结果时回退到首次随机结果 |

固定第一张地图开启时，“第一张地图选择方”会变为只读。固定地图选择框始终有合法默认地图，不允许保留空值。

交互随机时，两队分别提交 0 或 1；双方选择完成或 30 秒超时后计算 XOR，超时方按 0 处理。系统统一公布结果。

已选择过的地图在后续选图面板中以橙色边框标识，并且不可再次选择。地图名称在选图界面使用中文。

#### 攻防与阵营选择

选图确认后、上场成员确认前进入本步骤：

- 攻击/护送与运载目标模式必须点击“选择先进攻”或“选择先防守”。界面只明确当前选择队伍的决定，不再额外重复显示对手的相反职责。
- 对称地图仅在“对称地图允许选择阵营”开启时进入本步骤，选择“蓝色方”或“红色方”。
- 交互随机沿用双方提交 0/1 的 XOR 流程；系统随机由服务端认可的当前快照统一公布结果。
- 倒计时进度条只在当前操作方保持彩色；另一支队伍看到灰色进度条，以区分当前操作权。
- 选择完成后，主页地图卡片不额外堆叠攻防文字；本图分配会在比分录入页的“本图信息”中与禁用英雄一起展示。三方页面在下一次一秒轮询内进入上场成员步骤。

#### 地图选择超时

| 策略 | 处理方式 |
| --- | --- |
| 警告并延长30秒 | 警告当前选图方并重新给 30 秒 |
| 随机合法地图 | 从当前模式、地图池和不重复限制下的合法地图中随机选择 |
| 本小局判负 | 当前选图方本小局判负 |
| 交由管理员判断 | 停止自动推进，由管理员选择延长、随机合法地图或判负 |

### 3.4 上场成员设置

| 设置 | 行为 |
| --- | --- |
| 双方自由输入 `free_input` | 每队在本队高亮区域自由填写 2 输出、1 坦克、2 辅助；同队成员名不能重复 |
| 只使用预设成员 `preset_only` | 每个位置只能从该队预设名单中选择 |
| 跳过 `skip` | 不显示上场成员步骤，选图后直接进入英雄禁用或比分 |
| 预设成员 | 仅预设模式 | 为两队提供各自可选名单 |

队伍只能编辑本队阵容，管理员可以编辑两侧。某队确认后，另一队在确认自己的名单前只能看到“对方已确认”，不能看到具体成员；双方都确认后，后续禁用页面才展示完整的两队成员与职责。

点击“确认上场成员”后会显示确认窗口，列出本队（管理员为两队）的 5 个位置和成员。按钮统一为“返回修改”和“确认”，两者的宽度、高度和字号完全一致。

#### 上场成员超时

| 状态 | 结果 |
| --- | --- |
| 两队均未确认 | 不会同时判负；清空本轮未确认输入，重置双方状态，并重新给予完整的 `playerSelectSeconds` |
| 一队已确认、另一队未确认，策略为“警告并延长30秒” | 未完成方获得额外 30 秒 |
| 一队已确认、另一队未确认，策略为“本小局判负” | 未完成方本小局判负 |
| 一队已确认、另一队未确认，策略为“交由管理员判断” | 冻结选人步骤，管理员选择延长或判负 |

这个“双未确认重开”规则优先于判负策略，确保不会出现双方同时因选人超时被判负。

### 3.5 英雄禁用设置

| 设置 | 行为 |
| --- | --- |
| 启用禁用规则：否 | 跳过全部英雄禁用步骤，阵容确认后直接进入比分 |
| 启用禁用规则：是 | 每张地图进入先后手和英雄禁用流程 |
| 败者选择先后 `allow_loser_choose` | 地图选择方选择“选择先手”或“选择后手” |
| 败者固定先手 `loser_must_first` | 跳过先后选择，地图选择方直接成为先手 |
| 禁用首次先手方 | 第一张地图使用系统随机、交互随机、队伍1或队伍2；后续按地图规则推进 |

选择先手/后手按钮会立即确认，不额外弹出二级页面。若先后手选择倒计时结束，默认选择先手；这个超时不触发英雄禁用的判负策略。

选择具体英雄后会弹出确认窗口，明确列出被禁用英雄；按钮统一为“返回修改”和“确认”。

英雄卡片状态：

- 正常亮色：本轮可以选择。
- 灰色主体、橙色边框：本队此前地图已经禁用过该英雄，不能再次选择。
- 灰色主体、红色边框：对手在本轮已经禁用该英雄，不能选择。
- 已禁用英雄同时显示在对应队伍阵容标题旁；无禁用结果时不显示占位文本。

#### 英雄禁用超时

| 策略 | 处理方式 |
| --- | --- |
| 警告并延长30秒 | 警告当前操作方并重新给 30 秒 |
| 随机合法英雄 | 在角色、历史禁用和对手本轮禁用限制内随机选择 |
| 本小局判负 | 当前未完成英雄禁用的一方本小局判负 |
| 交由管理员判断 | 管理员选择延长、随机合法英雄或判负，也可以直接在英雄面板中代选 |

### 3.6 比分设置与暂停

| 设置 | 行为 |
| --- | --- |
| 管理员填写比分 `admin_only` | 只有管理员可以填写并确认双方比分 |
| 一方提交、另一方确认 `team_submit_opponent_confirm` | 任一队填写完整比分并提交；另一队确认或拒绝；拒绝后等待管理员处理 |

比分阶段包含两套独立的队伍暂停：

- 队伍1只能暂停/恢复队伍1；队伍2只能暂停/恢复队伍2。
- 管理员可以操作任意一方。
- 每一方同时显示“本小局累计暂停”和“本次暂停”。
- 每一方同时显示“本小局暂停次数”；每次从未暂停切换到暂停时加一。
- 恢复后，本次暂停归零，累计暂停保留到本张地图比分确认完成。
- 任一方处于暂停时，比分确认倒计时冻结。
- 两方同时暂停时，倒计时只冻结一次，但两方各自累计自己的暂停时长。
- 管理员全局暂停与队伍暂停不能同时新建；应先结束当前队伍暂停。
- 暂停开始时间、累计时长和次数由服务端计算；客户端只提交开始或恢复意图，不能覆盖计时结果。
- 任一队提交比分后，正在进行的队伍暂停立即结算并停止；比分等待对方确认期间不能再开始或恢复暂停。

比分相同视为平局，不增加任何一方系列赛胜图。非平局时，较高分一方增加一张系列赛胜图。

### 3.7 检查点

每张地图可以分别开启以下回退点：开始前倒计时、选择地图、选择上场成员、选择禁用先后、先手禁用、后手禁用、比分录入。

检查点只有在对应步骤已经到达或完成后才可用。管理员回退后，目标步骤之后的地图、阵容、英雄禁用和比分会按回退范围清理，并向两队显示管理员处理通知。队伍确认通知后继续。

## 4. 完整比赛流程

### 4.1 创建与赛前

1. 在首页创建房间，获得队伍1、队伍2、管理员和直播四个入口。
2. 管理员从当前模板开始、导入模板、恢复网站默认配置，或从 JSON 导入模板。
3. 管理员编辑并保存草稿，然后确认配置；若启用默认配置开始，则可直接开放队伍准备。
4. 两队进入各自入口；允许改名时先确认自己的队伍名称，再点击“是”完成准备。
5. 自动开始开启时，两队准备后直接开始；否则由管理员普通开始。紧急情况下管理员可强制开始。

### 4.2 每张地图

1. 可选的赛前/场间倒计时。
2. 确定选图方；第一张按首图策略，后续由最近非平局地图的败者选择。
3. 若首图固定或使用固定地图顺序，跳过对应选图；否则选择地图并二次确认。
4. 需要攻防或阵营选择时，由配置指定的一方确认本图分配。
5. 按阵容模式填写上场成员；双方分别二次确认。跳过阵容模式时直接进入下一步。
6. 若英雄禁用关闭，跳过禁用；否则确定先后手，再由先手与后手分别选择并二次确认英雄。
7. 进入比赛外部实际对局。
8. 在比分面板填写、提交和确认比分；等待过程中可使用每队独立暂停。
9. 记录本图结果；若未达到系列赛胜图数，进入场间休息并开始下一张。

### 4.3 比赛结束

1. 任一方达到 FT2/FT3/FT4 所需胜图数后立即结束系列赛。
2. 不再开启额外地图、阵容、英雄禁用或比分步骤。
3. 主页面保留最终地图结果、系列赛比分、连接状态和历史记录。

## 5. 二次确认规则

所有选择确认窗口只展示标题和具体选择内容，统一使用“返回修改”和“确认”，不显示“二次确认”或重复的继续提示。

| 操作 | 是否二次确认 | 确认内容 |
| --- | --- | --- |
| 选择地图 | 是 | 地图模式与中文地图名 |
| 确认上场成员 | 是 | 对应队伍的职责与成员清单 |
| 选择禁用英雄 | 是 | 英雄中文显示名 |
| 选择先手/后手禁用 | 否 | 点击“选择先手”或“选择后手”后立即生效 |
| 比分提交/确认 | 使用比分流程自身的提交与对方确认 | 双方比分 |

## 6. 管理员运行时功能

- 查看并操作所有队伍步骤。
- 普通开始、强制开始和回退赛前配置；比赛开始后管理员入口与检查点回退区直接显示在主页下方，无需快捷键展开。
- 全局暂停/恢复活动倒计时。
- 在比分阶段操作任意一队暂停。
- 对配置为“交由管理员判断”的超时执行延长、随机合法选择或本小局判负。
- 回退到已启用且已到达的检查点。
- 从房间模板、网站默认配置或 JSON 重建房间草稿。

所有配置保存、开始、回退、地图、阵容、英雄禁用、比分、暂停和超时操作都会写入房间快照及历史事件。运行时数据保存在 Docker 挂载的 `backend/data/runtime/`，普通构建、测试和重启容器不会主动删除这些数据。

## 7. 多端同步规则

- 队伍1、队伍2、管理员和直播入口每秒从服务端读取一次权威快照；检测到更高版本后立即按服务器当前阶段重绘页面。
- 连接状态心跳不会提前移动快照版本游标，因此不会再出现“心跳看见新版本、界面却跳过该版本”的情况。
- 服务端为比赛步骤计算单调阶段序号。旧页面提交的地图、阵容、禁用或比分快照如果会让比赛回到较早阶段，将收到冲突响应，并自动切回服务器当前页面。
- 双方可能同时提交的字段采用按入口合并：上场成员只合并本队阵容与准备状态，队伍暂停只合并本队暂停，交互随机只合并本队的 0/1。
- 未提交的上场成员和比分保存在当前浏览器本地草稿中；轮询更新对手状态时不会清空本地仍可编辑的输入。

- JSON Schema：`static/config/match-config.schema.json`

模板最外层包含 `schemaVersion`、`name`、可选 `description` 和 `config`。导入时服务端会校验字段、枚举、时间范围、地图池、模式数量、固定地图和赛制槽位；无效配置不会进入房间。
